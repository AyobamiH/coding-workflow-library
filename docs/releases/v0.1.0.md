# v0.1.0 Release Notes

Release date: 2026-06-19

## What This Library Is

`autonomous-coding-workflow-library` is a local autonomous coding workflow system. It packages reusable skill files, route metadata, evidence helpers, release checks, and a permission-gated runner for coordinating coding work without silently crossing into commits, pushes, releases, deploys, database changes, or production calls.

The CLI command is:

```bash
coding-workflow --help
```

## Main Capabilities

- Ledger-driven `scripts/run-next` workflow selection.
- Explicit permission gates for edits, commits, pushes, tags, releases, deployments, database mutation, secrets, and production checks.
- Reusable Markdown skill files under `skill-files/`.
- Route metadata and route audit via `routes/skill-routes.json` and `scripts/route-audit`.
- Skill hygiene and validation via `scripts/skill-cleaner` and `scripts/validate-skills`.
- Evidence-pack generation and failure-evidence classification.
- GitHub auth, exact-file commit, public source handoff, and PR lifecycle support.
- Supabase RLS, Edge Function deploy boundary, scheduler/Vault, runtime verification, and production handoff skills.
- Cloudflare deployment planning and Opstruth runtime-truth model support.
- Package readiness, release preflight, clean-temp portability, and local tarball install smoke.

## CLI Examples

```bash
coding-workflow --help
coding-workflow routes
coding-workflow validate
coding-workflow package-readiness --repo . --expect-package --expect-cli
coding-workflow release-preflight --repo . --mode cli
coding-workflow run-next --repo /path/to/repo --dry-run --allow <permission-flag>
```

## Safety Boundaries

External mutations remain permission-gated. This release does not grant permission to publish npm packages, run `npm version`, create GitHub releases, deploy, run Supabase or Cloudflare commands, call production endpoints, print secrets, force-push, or stage broad/excluded paths.

## Installation Status

This is a GitHub source and local tarball candidate release only.

NPM status: npm unpublished.

Use local tarball or GitHub source checks only until John approves a separate npm publication gate.

## Known Limitations

- Not every route has been production-tested across arbitrary repositories.
- The package name previously returned `E404 Not Found`, but ownership and future availability are not guaranteed until publication.
- GitHub release creation has not occurred.
- Product-specific Supabase, Cloudflare, and runtime lanes still require target-specific permission and evidence.
