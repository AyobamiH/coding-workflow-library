# v0.2.0 Release Notes

`v0.2.0` changes the autonomy model from route-by-route permission babysitting to objective-level authority.

The core rule is:

> The system requests authority for consequences, not permission for every tool call.

## What Changed

- Added objective authority classes for local execution, remote publication, production mutation, secret mutation, and destructive action.
- Added structured blocker states: `BLOCKED_CAPABILITY`, `BLOCKED_PERMISSION`, `BLOCKED_SAFETY`, `BLOCKED_DECISION`, `WAITING_CONDITION`, and `COMPLETE`.
- Extended lane state with an active objective, authority envelope, checkpoints, and blockers.
- Added CLI objective commands:

```bash
coding-workflow objective show --lane coding-workflow-library --state-file /path/to/lanes.json
coding-workflow objective approve --lane coding-workflow-library --grant remote_publication --state-file /path/to/lanes.json
coding-workflow resume --lane coding-workflow-library --state-file /path/to/lanes.json
```

- Updated `run-next` so child routes inherit authority from the selected lane objective.
- Preserved legacy `--allow` route flags for existing scripts and prompts.
- Added deterministic tests for authority inheritance, capability blockers, safety blockers, decision blockers, waiting conditions, checkpoint resume, idempotency, dry-run behavior, lane isolation, and secret-shaped state rejection.

## Safety Boundaries

Local file edits, tests, validation, package dry-runs, local commits, and read-only checks can proceed under `local_execution`.

External consequences remain protected:

- Git push, PR mutation, tag push, GitHub Release creation, and npm publication require `remote_publication`.
- Supabase, Cloudflare, scheduler, database, and production mutations require `production_mutation`.
- External secret creation or rotation requires `secret_mutation`.
- Force push, history rewrite, and deletion require `destructive_action`.

Capability is separate from authority. Missing npm authentication is recorded as `BLOCKED_CAPABILITY`, not as a request for permission.

## Installation Status

This is still an unpublished package candidate. Status: npm unpublished. Use local tarball or GitHub source testing only.

```bash
npm pack --dry-run --json
npm test
./bin/coding-workflow.js --help
```

## Known Limitations

- npm publication has not occurred.
- No GitHub Release has been created for this version.
- Some production-lane routes are proven for their original repositories and should be revalidated before reuse elsewhere.
- Remote publication is intentionally waiting for one objective-level authority grant.
