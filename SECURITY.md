# Security

Please report security issues privately through GitHub private vulnerability reporting if it is enabled for the repository. A dedicated security contact has not been configured yet.

Do not include secrets, credentials, tokens, database URLs, private logs, or sensitive evidence in reports. Describe the affected workflow, command, skill, or route, and include safe reproduction steps that avoid printing secret values.

This project controls automation through explicit permission boundaries. Unsupported or ambiguous actions must stop at a clear boundary instead of being inferred. Publishing, deployment, migration execution, remote secret mutation, GitHub release creation, and production endpoint calls all require separate approval.
